# Example Task Prompts

Use these prompts as examples of appropriate `paperplot-skills` tasks.

1. Use PaperPlotR to create a publication-ready boxplot from `data.csv`, comparing `condition` on the x-axis and `response` on the y-axis. Export PDF and PNG without overwriting old files.

2. Refine this existing ggplot2 figure into a manuscript-ready figure using PaperPlotR. Keep the original script and outputs unchanged, and create a new versioned output stem.

3. Create a 2x2 multi-panel figure using PaperPlotR. Use lowercase bold panel labels, collect the shared legend if readable, and export PDF plus PNG.

4. Generate a PCA scatter plot from `pca_scores.csv` with PC1 and PC2 on the axes, group-based colors, explained variance labels, and a vector output for manuscript editing.

5. Create a heatmap from long-form data with columns `feature`, `sample`, and `score`. Use a non-rainbow gradient palette and control the colorbar size.

6. Build a correlation scatter plot with `expression` on the x-axis and `phenotype` on the y-axis. Add a linear trend line and leave a placeholder for correlation statistics.

7. Create a grouped barplot with error bars from summary data. Use consistent colors and make sure category labels are readable.

8. Use PaperPlotR to convert this rough ggplot2 plot into a journal-style figure with consistent theme, color palette, export size, and notes.

9. Export a submission-ready figure as PDF, SVG, PNG, and TIFF where supported. Use versioned file names and write a notes file with file sizes and QA checks.

10. Check whether this figure legend matches the panel labels and order in my 2x2 figure. If it does not, create a corrected version without overwriting the original.

11. Create a violin plus dot plot for distribution comparison across groups, using a clean scientific theme and a notes file documenting any remaining readability issues.

12. Standardize colors across several panels using a user-provided semantic color mapping, then export a new manuscript-ready multi-panel figure.
